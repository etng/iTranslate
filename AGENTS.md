# 用户偏好记录

- 对话与文档统一使用中文。
- 项目文档（如 `README.md`、`PRD.md`、任务进展文档）要求中文。
- 代码注释、提交说明尽可能中文化。
- 开发过程中持续执行单元测试与 e2e。
- 代码需要保留可扩展接口，便于后续接入更多模型。
- 需要阶段性提交（小步提交）。
- Git 提交信息遵循 Conventional Commits，并使用约定 emoji。
- 文档中的 `cat` 示例命令优先追加 `| pbcopy`，便于直接复制。
- 保留根目录 README.md 和 AGENTS.md（前者是项目入口、后者是代理规范文件），把业务文档迁移到 docs/，确保 `docs/task-pool.md` 是用于勾选任务进度清单的任务池文档，以便我管理进度。
- 请尽可能用Makefile来组织散落在代码中各处的脚本命令，方便我快捷的操作，特别希望有一个一键初始化数据并把程序跑起来（web的话对应的服务，桌面程序的话就是对应的入口程序）
- 新需求处理时先拆分到任务池待办，再执行开发，完成后立即回填勾选状态。
- 任务池中完成的“本轮待办”应及时归档到“已完成”，避免长期堆积在本轮列表。
- 导出类功能优先提供“选择保存位置”，并允许设置默认路径与默认作者等偏好。
- 表格和紧凑导航中的操作偏好使用图标按钮，并通过 hover 文案（title）辅助识别。
